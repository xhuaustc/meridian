1. 布局结构重构

当前（Before）
decorations: true → 系统原生标题栏
+ 自定义 Titlebar 组件（引擎状态、启停、主题、语言）
= 双层标题栏，48px + 系统栏 ≈ 80px 浪费

侧边栏：纯色 #f5f5f4 背景
三平台完全相同的 Web 风格
改后（After）
macOS：隐藏原生标题栏，交通灯嵌入侧边栏顶部
Windows/Linux：保留原生标题栏，删除自定义 Titlebar

侧边栏纯导航，引擎状态移入内容区顶栏
主题/语言切换移入设置页
消除双层标题栏 → 单层

3. 组件层面改进

侧边栏 Sidebar
• 纯导航，移除所有非导航元素
• macOS：顶部留 52px 空间给交通灯 + 应用名
• Windows/Linux：顶部无需额外留空
• 选中项高亮跟随平台强调色
内容区顶栏 Toolbar
• 替代原 Titlebar 组件
• 左侧：页面标题
• 右侧：引擎状态 pill + 启停/重载 + 页面操作按钮
• macOS 作为窗口拖拽区
卡片与表格
• 增加微阴影提升层次感
• hover 行背景变化更明显
• 过渡动画 150-200ms ease
设置页
• 新增主题切换（从标题栏迁入）
• 新增语言切换（从标题栏迁入）
• 保留已有设置项

4. 技术实现要点

Rust 端
• macOS：decorations: false + titleBarStyle: "overlay" 或 transparent: true
• macOS：使用 objc2-app-kit（已有依赖）设置 NSVisualEffectView
• Windows：引入 window-vibrancy crate 启用 Mica
• 新增 IPC 命令 get_platform 返回 "macos" | "windows" | "linux"

前端
• 平台检测 hook：usePlatform() 返回当前平台
• CSS 层：[data-platform="macos"] / [data-platform="windows"] / [data-platform="linux"] 选择器
• AppShell 布局根据平台调整 grid（macOS 无系统标题栏行）
• 删除 Titlebar 组件，新建 ContentToolbar 组件
• 平台色彩 token 覆盖
不在本次范围内：自定义窗口控件（Windows/Linux 保留原生）· 系统强调色动态读取（使用固定平台色）· 侧边栏可拖拽调整宽度 · 动效系统（仅做基础 hover/transition）
