UI 专家评估

方案 A · 侧边栏底部
优势
• 顶部极简，视觉呼吸感强
• 引擎状态固定可见，不随页面切换消失
• 符合 macOS 侧边栏底部放全局控制的惯例（如 Finder "标签"）
劣势
• 引擎是高频操作，放底部视线距离远
• 如果导航项多，底部可能被挤压
方案 B · 内容区顶栏 ⭐ 推荐
优势
• 侧边栏职责单一（纯导航），认知负担最低
• 引擎状态在视线焦点区，操作便捷
• 跨平台一致性最好 — 三平台内容区布局相同
• 与 "添加代理" 等页面操作按钮同层，逻辑分组清晰
劣势
• 顶栏元素稍多（标题 + 引擎 + 按钮）
方案 C · 侧边栏顶部
优势
• 引擎状态一目了然，打开就看到
• 紧凑，不浪费空间
劣势
• 交通灯区域拥挤，与 macOS HIG 规范冲突
• 跨平台差异大 — Windows/Linux 没有交通灯，顶部布局完全不同
• 侧边栏职责混乱（导航 + 全局控制）
UI 专家结论：推荐 方案 B。核心理由是 职责分离 — 侧边栏只做导航，内容区顶栏承载页面上下文（标题、操作、状态）。这不仅符合 macOS HIG 的导航侧边栏规范，更关键的是三个平台可以共享内容区布局代码，只有侧边栏和标题栏需要平台适配，大幅降低维护成本。同时，主题切换和语言切换属于低频设置操作，应移入设置页面，减少常驻 UI 的噪声。
